# ПЛАН: Быстрое обучение модели жокеев перед каждым забегом

## Проблема

Сейчас модель знает только 3 цвета (green/red/yellow). Нужно:
- Перед каждым забегом сфоткать каждого жокея (15 фото с телефона)
- Быстро обучить модель (< 5 минут)
- Загрузить в пайплайн и сразу видеть результат

## Архитектура решения

### Шаг 1: Python-песочница для экспериментов (без DeepStream)

**Цель**: Работать с ОДНИМ видео, видеть каждую детекцию, отлаживать модель.

Создать `tools/sandbox.py` — интерактивный инструмент:

```
python tools/sandbox.py video.mp4
```

Что делает:
- Открывает одно видео (не 25)
- YOLOv11s детекция (PyTorch, не TensorRT)
- Показывает КАЖДУЮ детекцию крупно: bbox, торс-кроп, классификация
- Кнопки: пауза, покадровый, перемотка
- Сохраняет кропы торсов в папку для дообучения

По сути — улучшенная версия `tools/test_visual.py`, но с фокусом на:
- Крупное отображение кропов (не мелкие bbox на полном кадре)
- Автосохранение кропов в `data/crops/{race_name}/unsorted/`
- Confidence-метрики для каждой детекции

**Уже есть**: `tools/test_visual.py` — его можно взять за основу.

---

### Шаг 2: Фото жокеев с телефона → датасет

**Workflow перед забегом:**

```
data/races/2026-03-15_race1/
├── jockey_01_ivanov/     ← 15 фоток с iPhone
│   ├── photo_01.jpg
│   ├── photo_02.jpg
│   └── ...
├── jockey_02_petrov/
│   └── ...
├── jockey_03_sidorov/
│   └── ...
└── ...
```

Создать `tools/prepare_dataset.py`:
- Берёт фотки жокеев (полные фото, не кропы)
- Запускает YOLOv11s → находит человека
- Вырезает торс-кроп (те же параметры: 10-40% высоты, -20% по краям)
- Раскладывает в структуру для обучения:

```
data/races/2026-03-15_race1/torso_crops/
├── jockey_01_ivanov/
│   ├── crop_001.jpg
│   └── ...
├── jockey_02_petrov/
│   └── ...
└── ...
```

Также добавляет аугментации (яркость, контраст, поворот) чтобы из 15 фоток
сделать 150+ кропов.

---

### Шаг 3: Быстрое обучение (< 5 минут)

Создать `tools/quick_train.py`:

```
python tools/quick_train.py data/races/2026-03-15_race1/torso_crops/
```

Архитектура: **EfficientNet-V2-S** (уже есть в `train_color_classifier.py`):
- Freeze backbone (ImageNet pretrained)
- Обучаем только head: 3-5 эпох
- 15 фоток × 10 аугментаций = 150 сэмплов на класс
- На GPU (RTX 3090/4090) = 2-3 минуты

Выход:
- `models/race_classifier.pt` (PyTorch)
- `models/race_classifier.onnx` (ONNX)
- Автоматический экспорт в TensorRT:
  `models/race_classifier.engine`

**Важно**: Классы будут динамические (не 3 цвета, а N жокеев).
Маппинг `class_id → jockey_name` сохраняется в JSON рядом с моделью.

---

### Шаг 4: Проверка на видео (Python, без DeepStream)

```
python tools/sandbox.py video.mp4 --model models/race_classifier.pt
```

Видишь на экране:
- Bbox каждого жокея
- Имя жокея (не "green", а "Иванов")
- Confidence
- Torso-кроп крупно

Если плохо — добавляешь ещё фоток, перезапускаешь quick_train.

---

### Шаг 5: Деплой в DeepStream (когда модель готова)

Когда модель даёт хороший результат в Python-песочнице:

1. Конвертация ONNX → TensorRT engine (уже есть в `export_trt.py`)
2. Обновление маппинга классов в `config.h`:
   - Вместо `COLOR_BLUE/GREEN/...` → динамический список жокеев
   - Маппинг читается из JSON при запуске
3. Перезапуск Docker-контейнера с новой моделью

Файлы которые нужно изменить в C++:
- `config.h` — динамическое количество классов (не фиксированное NUM_COLORS=5)
- `color_infer.h/cpp` — читать маппинг из JSON, убрать хардкод MODEL_TO_SHM
- `pipeline.cpp` — OSD labels показывают имена жокеев

---

### Шаг 6: Hot-reload без пересборки Docker

Сейчас workflow: изменил модель → пересобрал Docker → перезапустил.
Нужно: заменил .engine файл → перезапустил только DeepStream процесс.

Решение:
- models/ уже монтируется как volume в docker-compose (`./models:/app/models:ro`)
- Добавить API endpoint: `POST /api/reload-model`
- Python-сервер перезапускает C++ subprocess с новым engine

## Альтернативный подход: Embedding (без переобучения)

Вместо классификации можно использовать **embedding-подход**:

1. Модель (ResNet/EfficientNet) выдаёт вектор 128-256 размерности
2. Перед забегом: фоткаешь жокеев → получаешь эталонные вектора
3. Во время забега: для каждого детекции считаешь cosine similarity
4. Кто ближе — тот и жокей

**Плюсы**: Не нужно обучать вообще. Просто фоткаешь → готово.
**Минусы**: Менее точно при похожих формах. Сложнее деплоить в TensorRT.

Рекомендация: **начать с fine-tuning** (Шаги 1-6), потому что:
- Инфраструктура обучения уже есть (`train_color_classifier.py`)
- EfficientNet с 15 фотками обучается за 2 минуты
- Embedding потребует больше R&D

## Как экспериментировать (ответ на главный вопрос)

**Правило**: Всю отладку делаем в Python. DeepStream — только для продакшна.

```
ОТЛАДКА (Python):
  iPhone фото → prepare_dataset.py → quick_train.py → sandbox.py (видео)
                    ↑                                      ↓
                    └── не нравится? добавь фоток ──────────┘

ПРОДАКШН (DeepStream):
  Модель готова → export_trt.py → docker restart backend
```

DeepStream НЕ нужен для экспериментов. Используй `tools/sandbox.py` с одним видео.
Когда модель работает хорошо на Python — тогда деплой в C++/DeepStream.
